# practice01 目录代码实验报告

## 1. 实验目的

- 了解和实现基于 HTTP 协议的 LLM (大语言模型) API 调用
- 掌握流式和非流式两种不同的 API 调用方式
- 学习环境变量配置和管理
- 实现简单的命令行聊天客户端
- 分析和统计 LLM 响应的性能指标

## 2. 实验过程

### 2.1 代码结构分析

practice01 目录包含两个 Python 文件：
- `chat_client.py`：实现流式调用 LLM API 的聊天客户端
- `llm_client.py`：实现单次调用 LLM API 并统计性能指标

### 2.2 核心功能实现

#### 2.2.1 环境变量管理
两个文件均实现了 `load_env()` 函数，用于从 `.env` 文件加载配置：
- 读取并解析 `.env` 文件
- 将配置项设置为环境变量
- 检查必要的配置项是否存在

#### 2.2.2 API 调用实现

**流式调用（chat_client.py）**：
- 构建 HTTP/HTTPS 请求
- 设置 `stream: true` 参数
- 实时处理并输出服务器返回的数据流
- 维护完整的聊天历史

**非流式调用（llm_client.py）**：
- 构建 HTTP/HTTPS 请求
- 发送完整请求并等待响应
- 计算执行时间和 Token 使用情况
- 显示详细的性能统计信息

#### 2.2.3 命令行交互
`chat_client.py` 实现了简单的命令行交互界面：
- 循环获取用户输入
- 流式显示 LLM 响应
- 支持通过 Ctrl+C 退出

## 3. 实验结果

### 3.1 功能验证

**chat_client.py**：
- 成功加载环境变量
- 实现了连续的聊天对话
- 流式输出 LLM 响应，提升用户体验
- 正确维护聊天上下文

**llm_client.py**：
- 成功调用 LLM API
- 准确计算并显示执行时间
- 正确统计 Token 使用情况
- 提供详细的性能指标

### 3.2 性能分析

通过 `llm_client.py` 的测试调用，可以得到以下性能指标：
- 执行时间：从发送请求到收到响应的总时间
- Token 消耗：包括提示 Token 和完成 Token
- Token 处理速度：每秒处理的 Token 数量

这些指标有助于评估 LLM API 的性能和成本。

## 4. 实验总结

### 4.1 实现效果

本次实验成功实现了两个 LLM 客户端：
1. **聊天客户端**：提供了交互式的聊天体验，支持流式输出，用户可以进行多轮对话
2. **单次调用客户端**：提供了详细的性能统计，适合测试和分析 API 性能

### 4.2 技术要点

- **HTTP 客户端**：使用 `http.client` 库实现 HTTP/HTTPS 请求
- **流式处理**：通过逐行读取响应实现流式输出
- **环境变量管理**：从 `.env` 文件加载配置，提高代码可移植性
- **错误处理**：实现了基本的错误检测和提示
- **性能统计**：计算并显示关键性能指标

### 4.3 应用场景

- **chat_client.py**：适用于需要多轮对话的场景，如聊天机器人、客服系统等
- **llm_client.py**：适用于单次查询、性能测试和成本分析场景

3. 实现更丰富的命令行交互功能

通过本次实验，我们不仅实现了基本的 LLM API 调用功能，还学习了流式处理、环境变量管理等技术点，为后续更复杂的 LLM 应用开发奠定了基础。
