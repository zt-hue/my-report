# practice04 目录代码实验报告

## 1. 实验目的

- 扩展工具调用客户端功能
- 实现聊天历史搜索功能
- 学习聊天历史总结和关键信息提取
- 提升系统的上下文处理能力
- 构建更加智能的AI助手

## 2. 实验过程

### 2.1 代码结构分析

practice04 目录包含两个文件：
- `tool_client.py`：扩展了工具调用功能的客户端
- `prompt04.txt`：更新后的系统提示词配置文件

### 2.2 核心功能实现

#### 2.2.1 新增工具函数

**聊天历史搜索工具**：
- `search_chat_history`：搜索聊天历史记录，当用户输入以'/search'开头或表达'查找聊天历史'的意思时使用
- 从log.txt文件中读取聊天历史
- 构建搜索提示，调用LLM进行相关信息提取

#### 2.2.2 新增功能模块

**聊天历史总结**：
- `summarize_chat_history`：总结聊天历史，压缩前70%的内容，保留最后30%的原文
- 过滤掉系统消息，只保留用户和助手的消息
- 调用LLM进行总结
- 构建新的聊天历史，保留系统提示和总结结果

**关键信息提取**：
- `extract_key_info`：提取聊天关键信息，按照5W规则（谁Who、做了什么事What、什么时候When、在何处Where、为什么要做这个事Why）
- 过滤掉系统消息和总结消息
- 调用LLM进行关键信息提取
- 将提取的关键信息写入log.txt文件

#### 2.2.3 工具调用流程优化

- 增加了对聊天历史搜索的特殊处理
- 实现了聊天历史长度和轮数的监控
- 当聊天历史过长时自动进行总结
- 定期提取关键信息并保存

#### 2.2.4 命令行交互增强

- 增加了对'/search'命令的支持
- 提供了更友好的用户提示和示例
- 显示聊天历史总结和关键信息提取的进度

## 3. 实验结果

### 3.1 功能验证

**聊天历史搜索功能**：
- 成功搜索聊天历史记录
- 能够根据用户查询提取相关信息
- 支持多种搜索触发方式（'/search'命令或自然语言表达）

**聊天历史总结功能**：
- 成功总结聊天历史，压缩冗长内容
- 保留最近的聊天内容，确保上下文连续性
- 生成的总结准确反映聊天内容

**关键信息提取功能**：
- 成功按照5W规则提取关键信息
- 提取的信息准确反映聊天内容的核心要点
- 能够将提取的信息写入log.txt文件

**系统稳定性**：
- 实现了更完善的错误处理，包括API响应解析错误的处理
- 系统能够稳定运行，处理各种用户输入

### 3.2 性能分析

- **响应速度**：聊天历史总结和关键信息提取会增加一定的响应时间，但提升了系统的上下文处理能力
- **内存使用**：由于定期总结聊天历史，内存使用保持在合理范围
- **存储使用**：关键信息提取功能会增加log.txt文件的大小，但提供了有价值的信息存储

## 4. 实验总结
